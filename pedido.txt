Pode ser backend em JS
Frontend em JS
Não precisa de banco de dados nesse momento, somente processar o pedido e retornar ao front

Cardápio digital

- Tela 1
    - Informações dos restaurantes
    - Listagem de produtos genéricos
        - Listagem de item
        - Complementos
- Tela 2
    - Carrinho
    - Alterar quantidade
- Tela 3
    - Tela de informações de usuários
        - Nome
        - Telefone
- Tela 4
    - Tela de pagamentos
        - Dinheiro e cartão de crédito, débito
        - Pagamentos apenas para pagar na entrega
- Tela 5
    - Tela de sucesso
        - Mostrar as principais informações do pedido
- Tela 6
    - Tela de acompanhamentos de pedidos

- Tela 7
    - Tela de Histórico de pedidos
    - O cliente poderá visualizar pedidos já realizados, coletando pelos IDs salvos no localstorage

Tela de avaliações
    - Após realizar o pedido o cliente poderá avaliar o pedido realizado, não necessita ser um pedido entregue

Gestor de pedido
    - É onde o cliente recebe os pedidos e consegue realizar o fluxo dos status do pedido

# Requisitos
- Necessário realizar o pedido
- Calcular itens corretamente
- Precisa ser um fluxo linear de telas com possibilidade de voltar etapas também
- Ao finalizar o pedido, deve salvar o ID do pedido no localstorage, então assim o cliente consegue coletar posteriormente
